How to compress your images to optimize your website

We've all been in this situation at one point or another of our career as front-end developers. You finish your website and get it hosted on a server. You're excited to show it off to your friends and family, but when you send them the link, they tell you that the website is slow. You check the website on your phone and it's still slow. You check it on your computer and it's still slow. You check it on your smart toaster and it's still slow! So what gives? Why is your website so slow? Besides the fact that your smart toaster probably isn't meant for navigating the web, it could be a lot of things. It could be that your server is slow, or that your website is too big, or that your website is poorly optimized. In this article, we will focus on the last one, and learn how to optimize images for your website.

First thing first, instead of relying on our friends and families to judge the speed of our website, we can use a tool called Lighthouse. Lighthouse is a tool that is built into Chrome DevTools. It can be used to analyze the performance of our website. To use it, we need to open up Chrome DevTools and click on the Lighthouse tab. Once we do that, we can click on the Generate report button and Lighthouse will analyze our website and give us a report. Here's what the report looks like:

Yikes! As we can see, our website is not doing so well. Luckily for us, we can use Lighthouse to figure out what's wrong. If we scroll down to the bottom of the report, we can see that Lighthouse has identified the issues that are slowing down our website. In this case, it's the images. The images are taking up a lot of space and are slowing down our website. This looks like a job for Squoosh! Squoosh is a tool that can be used to compress images. It can be used to reduce the size of images without losing any quality. To use it, we can go to the Squoosh website and drag and drop our images into the tool.

Once we do that, if we check the bottom left side of the screen we should be able to see the original size of our image file. In my case I using an image that is 1.66 mbs in size. That's huge for an image! Normally I like to keep my images under 20 kb unless they are really detailed. To accomplish this, look to the right side of the screen. Here we will see several options to modify our image. We can change the quality, the dithering, the quantization, and the encoding. We can also change the dimensions of our image. Let's look at the resize option.

All we need to do is type in the dimensions we want our image to be. The original size of my image is 1024 x 1024. I'll reduce the size to 400 x 400. This will reduce the size of my image without losing any quality. Already we can see a huge difference in the image size, but we can do even better.

Another thing we can do to reduce the image size is to select another image type from the compress option. So how do you know what image type to use? Well, it depends on what you're using the image for. In general, the most commonly used image formats for web use are JPEG, PNG, and GIF. JPEG is a good choice for photographs and images with lots of colors, as it has good compression and supports lossy compression. PNG is a good choice for images with transparent areas or images with large blocks of solid color, as it supports lossless compression and has good transparency support. GIF is a good choice for simple graphics and animations, as it supports animation and has good transparency support. Another great format to use is WebP. WebP is a relatively new image format that is supported by most modern browsers. It's also a great choice for images with lots of colors, as it has good compression. Optimizing images in WebP format can significantly reduce the size of your webpage and improve its performance, especially on mobile devices.

I'll use WebP for this example. Simply select WebP from the compress option. Below that, you should see a slider that allows you to adjust the quality of your image. I'll move it around until I get a good balance between quality and size.

That's it! Now we can download our image and use it on our website. If we check the bottom right side of the screen, we can see that the size of our image has been reduced to 20 kb. That's an improvment of 99%! Compressing the image by 99% must mean that it's going to look terrible, right? Well, let's find out. Here's the original image vs the compressed image:

Not bad! As you can see, the image still looks great. The only difference is that the compressed image is a lot smaller. This means that our website will load faster and our users will have a better experience. Now that we know how to optimize images, let's go back to Lighthouse and see if our website is any faster.

Now that's what I call speed! There are a lot of other things we can do to improve our website's performance, but this is a great start! Many modern frameworks have packages that can handle image optimization for you. For example, if you're using Next.js, you can use the Image Optimization feature However, if you're using a framework that doesn't have this feature, you can use a tool like Squoosh to optimize your images.

So are there any downsides to optimizing images? Well, yes. If you're using an image that has a lot of detail, you may notice that the image looks a little blurry after you compress it. Also remember when I mentioned that Webp is a relatively new image format? Well, that means that not all browsers support it. If you're using an image that is only supported by a few browsers, you may want to use a different image format. Another option is to use a fallback image.
